AJAX: Asynchronous JavaScript And XML JSON: JavaScript Object Notation

JavaScript Object Notation, mejor conocido en el mundo del desarrollo web, como JSON, es un formato alternativo de envío y recepción de datos, es decir remplaza a XML o el envío de texto plano.

JSON utilza objetos literales y arrays literales como modelo de datos

            Array literal:

            var usuario=["juan","26"];
            alert(usuario[0]); //Imprimimos el primer elemento del array
            alert(usuario[1]); //Imprimimos el segundo elemento del array

Los elementos de un array literal se encierran entre corchetes y los valores contenidos van separados por coma. Cuando se define un array literal no se indica a cada elemento de que tipo se trata, podemos almacenar cadenas (entre comillas), números, valores lógicos (true,false) y el valor null. Para acceder a los elementos de un array literal lo hace por su nombre y entre corchetes se indica que elementos queremos acceder

            Objeto literal:

            var persona={
                "nombre":"juan",
                "clave":"xyz",
                "edad":"26"
            };

            alert(persona.nombre);  //Imprime el valor de la propiedad nombre del objeto persona.
            alert(persona['nombre']);

La sintaxis de JSON difiere levemente de lo visto anteriormente:

            {
                "nombre":"juan",
                "edad":"22",
                "estudios":["primario","secundario"]
            }

En la sintaxis JSON no aparecen variables, sino directamente se indica entre llaves las propiedades y sus valores.

También se ha eliminado el (;) luego de la llave final. El resto es igual.

La diferencia entre JSON y XML

            Objeto XML:
            < persona>
                < nombre>juan< /nombre>
                < edad>22< /edad>
                < estudios>
                < estudio>primario< /estudio>
                < estudio>secundario< /estudio>
                < /estudios>
            < /persona>

            Objeto JSON:
            {
                "nombre":"juan",
                "edad":"22",
                "estudios":["primario","secundario"]
            }

Se puede ver que es mucho más directa la definición de esta estructura.

Si tenemos que transmitir estas estructuras por internet la notación JSON es más liviana.

Otra ventaja es como recuperamos los datos en el navegador, como vimos si se trata de un archivo XML llamamos al método requestXML y luego accedemos por medio del DOM

En cambio con JSON al llegar el archivo procedemos a generar una variable en JavaScript que recree el objeto literal, esto mediante la el método parse del objeto JSON:

            var persona=JSON.parse(conexion1.responseText);
